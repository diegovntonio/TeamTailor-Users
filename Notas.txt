Proceso de programación
1. Desarrollo inicial: Manejo de errores básicos para no tener errores mudos durante el desarrollo
2. Refactorización a producción: Aquí se pulen los mensajes, se agrega logging y se separan las excepciones


Metodologias para mejorar la programacion
1. Metodologia "Top-Down": Para esta metodologia se empieza escribiendo el "esqueleto" del programa en la 
funcion main()
Como aplicarlo: Se debe escribir primero las llamadas a funciones que aun no existen luego se debe ir rellenando
estas funciones

2. Desarrollo basado en contratos (definir inputs/outputs): Antes de programar la logica interna de las
funciones, se debe definir que recibe y que entrega. En Python se usa Type Hinting para esto

3. Programacion incremental y atomica: Dividir el progreso en hitos. 
Ej: Hito 1: Lograr que la API imprima 1 solo usuario en consola.
Hito 2: Lograr que ese usuario se guarde en SQL
Hito 3: Implementar el while para que recorra todas las páginas
Hito 4: Agregar el try-except para los duplicados.
Regla de oro: Nunca pasar al hito 2 si el hito 1 no es solido
